Testing Workgroup Charter

1. Testbed improvement.

  • support sonic device as a neighbor.
  • include pdu/console information.
  • using exabgp to build bgp routes.
2. Pytest framework enhancement
  • How to execute commands on arista devices?
  • How to improve the DUT reboot? Add console connection support so that we can see the console log while rebooting.
  • Loganalyzer improvement can be implemented as a pytest fixture now.
3. Pytest conversion from ansible test.
  • strategy to conversion and deprecating the existing ansible test one-by-one. once a pytest has been developed, then we can deprecate the ansible test by calling the pytest in ansible playbook so that there is no change on the user side. Meanwhile, if there will be further improvement the improvement will go to pytest, and community does not need to maintain both versions.
4. Specific/new tests:
  • Platform related tests. ()
  • Qos/RDMA tests.
  • image upgrades testing.
  • acl tests improvement.
  • scalability tests on all the tables (l3/acl/fdb), error handling. for example, l3 route exceeds the asic capability.
  • validation checks, error scenario. for example, port-channel configuration sequence.
  • scenario, flap the link while sending the traffic.
  • test categorization: functional/scalability/negative tests/control plane performance/stress
  • test creation based on current outstanding issues (prevent regression)
5. Visualized DUT with data plane capabilities
  • adding specific data plane functions to the virtual switch.
6. One(AKA, Crystalnet) integration (open source)
